# Project Configurations

## nagi.yaml

The project configuration file. Generated by [`nagi init`](./cli.md#init) and placed at the project root (same level as `resources/`).

```yaml
backend:
  type: local
stateDir: ~/.nagi
defaultTimeout: 1h
notify:
  slack:
    channel: "#nagi-alerts"
export:
  connection: my-bigquery
  dataset: my_project.nagi_logs
terminationGracePeriodSeconds: 300
```

`stateDir` specifies the path to the Nagi state directory. `cache/`, `locks/`, `suspended/`, `logs.db`, `logs/`, and `watermarks/` are placed in this directory. Defaults to `~/.nagi` when omitted.

Setting `notify.slack` enables Slack notifications. Omitting it disables the notification feature. See [Notifications](../architecture/notifications.md) for configuration details.

Setting `export` enables exporting execution logs to a data warehouse. Compile auto-generates export resources and creates tables in your data warehouse. See [Export](../architecture/export.md) for details.
